# TAf Accessibility Mobile Classroom

## When to use

Use for student forms, support pages, videos, and any phone-first classroom UI.

## Inputs to read

- `AGENTS.md`
- relevant templates, CSS, and student-facing copy

## Rules

- Keep buttons large and touch-friendly.
- Use explicit labels and short messages.
- Make the 2-digit identifier requirement obvious.
- Preserve strong readable contrast.
- Keep navigation simple.
- Make confirmation and duplicate-submission messages easy to understand.
- Avoid technical jargon on student pages.

## Safe workflow

1. Review the page at narrow mobile widths first.
2. Check action labels, error states, and confirmations.
3. Keep one-screen decisions simple where possible.
4. Prefer clarity over decorative complexity.

## Stop conditions

- student wording is technical or ambiguous
- mobile interaction becomes cramped or confusing
- duplicate or confirmation feedback is unclear

## Acceptance checklist

- Touch targets are large enough.
- Labels and messages are clear.
- ID format guidance is obvious.
- Student navigation stays simple.
- No technical jargon leaks into the student flow.
