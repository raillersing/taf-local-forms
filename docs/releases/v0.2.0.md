# TAf Local Forms v0.2.0

## Résumé

Modules 2, 3 et 4 disponibles dans l'application locale terrain.

- **F020 — Page d'accueil avec choix étudiant / formateur** : la racine `/` affiche deux cartes de choix : « Je suis étudiant » (→ `/modules/`) et « Je suis formateur » (→ `/dashboard/`)

- **F020 — Blocs pédagogiques par module** : présentations intégrées (`module_X_pedagogy.html`) détaillant les objectifs, le contenu et le déroulement de chaque module
- **F021 — Contenu réel des présentations PowerPoint intégré dans l'espace étudiant** :
  - "Contenu réel des présentations PowerPoint intégré dans l'espace étudiant"

  - "Module 3 : Recherche efficace — formule des bons mots-clés (matière + notion + niveau + type), 4 étapes question→mots-clés→résultats→choix, outils de recherche, erreurs fréquentes"
  - "Module 4 : Sources fiables — méthode des 5 questions (QUI ? QUOI ? QUAND ? OÙ ? POURQUOI ?), indices auteur/date/preuves, réflexes à retenir"
  - "Nouvelles classes CSS : pedagogy-card, pedagogy-grid, pedagogy-pill, pedagogy-steps, pedagogy-warning, pedagogy-formula, pedagogy-tip, pedagogy-retention"

## Limites connues

- L'accès depuis un téléphone dépend de la qualité du Wi-Fi, du pare-feu Windows et de l'absence d'AP isolation.
- `TAF_LAN_HOST` doit être configuré dans `.env` pour figer l'adresse recommandée si l'IP change entre les sessions.
- Graphviz est optionnel pour générer les SVG d'architecture.
- Le port exposé par défaut est `8010` (modifiable via `TAF_HOST_PORT`).
- Le compteur de présence temps réel est indicatif : un élève qui ferme brutalement le navigateur disparaît après ~60s.
- Les changements réseau depuis `/dashboard/settings/` nécessitent un redémarrage Docker pour prise d'effet.
- Le panneau de configuration est réservé aux utilisateurs staff/superuser.
