# TAf Local Forms v0.2.0

## Résumé

Modules 2, 3 et 4 disponibles dans l'application locale terrain.

### Nouvelles fonctionnalités

- **Module 2** — Comprendre Internet (questionnaire complet)
- **Module 3** — Recherche efficace (questionnaire complet avec quiz à 7 questions)
- **Module 4** — Sources fiables (questionnaire complet avec auto-évaluation, todo, quiz, activité pratique)
- **Accueil public** (`/`) — liste des modules actifs
- **Cockpit formateur** (`/dashboard/`) — centralise tous les outils
- **Dashboards par module** (`/dashboard/module-2/`, `/module-3/`, `/module-4/`) — statistiques, filtres, tableau des réponses
- **Dashboard Module 4** — inclut un résumé des décisions (fiable / douteuse / pas encore)
- **Exports CSV** par module (`/dashboard/export/module-X.csv`)
- **Page accès réseau** (`/dashboard/network/`) — diagnostic IP, toutes les URLs M2/M3/M4, port, alertes
- **Configuration réseau sécurisée** (`/dashboard/settings/`) — modification de TAF_HOST_PORT, TAF_LAN_HOST, ALLOWED_HOSTS, CSRF_TRUSTED_ORIGINS, TIME_ZONE depuis l'interface (staff only, secrets jamais exposés)
- **Compteur temps réel** — présence élèves en cours de saisie affiché dans le cockpit formateur (heartbeat HTTP 15s étudiant, polling 8s dashboard)
- **Manuel terrain** (`docs/FICHE_DEMARRAGE_TERRAIN.md`) — pas à pas pour le formateur
- **Workflow agents et skills** — framework Codex orchestrateur
- **Graphify** — visualisation d'architecture (optionnel, nécessite Graphviz)
- **Nettoyage sûr** — scripts `taf-clean-status`, `taf-clean-after-merge`, `taf-docker-safe-clean`

## Limites connues

- L'accès depuis un téléphone dépend de la qualité du Wi-Fi, du pare-feu Windows et de l'absence d'AP isolation.
- `TAF_LAN_HOST` doit être configuré dans `.env` pour figer l'adresse recommandée si l'IP change entre les sessions.
- Graphviz est optionnel pour générer les SVG d'architecture.
- Le port exposé par défaut est `8010` (modifiable via `TAF_HOST_PORT`).
- Le compteur de présence temps réel est indicatif : un élève qui ferme brutalement le navigateur disparaît après ~60s.
- Les changements réseau depuis `/dashboard/settings/` nécessitent un redémarrage Docker pour prise d'effet.
- Le panneau de configuration est réservé aux utilisateurs staff/superuser.
